    • 2) SMART BUILDING

    A) What does smart mean?

  • Something that operates effectively and integrate into a communication network while concealing all complexity from the user to minimise human effort.

    • B) What are the main features needed?

  • use of sensors - temperature sensor, motion sensor, richter scale.
  • detectors which transmits the data to management system - cloud storages, API integration, real-time database.
  • System integration platform - API integration, MQTT, Firebase, AWS.
  • connection of people and technologies - management centres, security system .

    • C) What are some smart Building scenarios?

  • It employs preventative and predictive maintenance strategies and uses real-time field data to spot problems.
  • Sustainability and reuse of resources.
  • Smart evacuation system.
  • Voice control operation in room.
  • AI powered assistance.
  • Retina recognition door locks

    • 3) CONTROL INTERFACE

    A) What are the high level features?

  • Detection of any motion and sending data to control room giving indication that someone has arrived/leave
  • recognition and acting accordingly to the users.
  • recognition for door locking.
  • of any danger through temperature/richter scale and indication through alarms.
  • Speed and Efficiency with proper power backup system.

    • B) What are the parts needed?

  • Microcontrollers - Arduino, RaspberryPi, Motion Sensor
  • Sensors - Temperature, RFID
  • database.
  • integration.
  • User Interface.
  • recognition sensor/ voice recognising module.
  • level sensor (which will detect the tsunami water level for evacuation from the building).
